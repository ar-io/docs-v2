# Tagging Data

Tags are key-value pairs that you attach to your Arweave transactions to organize, categorize, and make your data discoverable. Think of them as metadata that describes what your data contains, who created it, when it was uploaded, and how it should be categorized. This tagging system is what makes the permaweb searchable and organized, transforming it from a collection of random data into a structured, queryable database.

The tagging system is incredibly flexible and powerful. You can add any number of tags to describe your data, from basic information like content type and title to more specific details like author, category, or custom attributes. Common tags include `Content-Type` (to specify the file type), `Title` (for a human-readable name), `App-Name` (to identify which application created the data), and `App-Version` (for version tracking). You can also create custom tags for your specific use case.

Tags are essential for making your data discoverable through search and query systems. When you upload data with proper tags, it becomes searchable through AR.IO gateways and other permaweb search tools. This means people can find your content by searching for specific terms, categories, or attributes. For example, if you tag a document with `Type: "blog-post"` and `Topic: "blockchain"`, it will appear when someone searches for blockchain-related blog posts.

The tagging system also enables powerful querying capabilities through GraphQL. You can search for data based on any combination of tags, making it easy to build applications that discover and interact with specific types of content. This is particularly useful for building decentralized applications that need to find and display content from multiple sources, or for creating curated collections of related data.
