# Why Run Your Own Bundler Extension?

Running your own bundler extension alongside your AR.IO gateway provides significant advantages for data submission and processing. Bundlers handle the complex process of batching multiple transactions into efficient bundles, reducing costs and improving upload performance.

## Performance Benefits

Bundlers can dramatically improve upload speeds by batching multiple data items together and processing them in parallel. This is especially beneficial for applications that need to upload large amounts of data or many small files simultaneously.

## Cost Optimization

By bundling transactions, you can reduce the overall cost of data submission to Arweave. Bundlers optimize the batching process to minimize fees while maximizing throughput, making them essential for cost-effective data operations.

## Custom Processing

Your own bundler allows you to implement custom processing logic, filtering, and validation before data reaches the Arweave network. This gives you control over what gets uploaded and how it's processed.

## Integration Benefits

Running a bundler alongside your gateway creates a complete data infrastructure stack. Users can submit data through your bundler and immediately access it through your gateway, providing a seamless experience and reducing dependency on external services.

This setup is particularly valuable for applications that need reliable, fast data submission with immediate availability and custom processing capabilities.
