Let's say you have some off-chain IP (ex. a book, a character, a drawing, etc). In order to register that IP on Story Protocol, you first need to mint an NFT. This NFT is the **ownership** over the IP. Then you **register** that NFT on Story, turning it into an IP Asset.

This tutorial enhances the standard Story Protocol workflow by:
1. Uploading your IP assets to Arweave for permanent, decentralized storage
2. Registering the assets with AR.IO names for easy access
3. Connecting the Arweave transaction IDs to your Story Protocol IP assets

## Prerequisites

Before starting the tutorial, make sure you have:

1. Completed the [AR.IO SDK Setup](/ar-io-sdk/getting-started)
2. Completed the [Story Protocol TypeScript SDK Setup](https://docs.storyprotocol.xyz/developers/typescript-sdk/setup)
3. Node.js installed (version 16 or higher)
4. An Arweave wallet file (JWK format)
5. A wallet with ETH on the Sepolia testnet for Story Protocol interactions

Create a new project directory and initialize a Node.js project:

```bash
mkdir arweave-story-integration
cd arweave-story-integration
npm init -y
```

Install the required dependencies:

```bash
npm install @ar.io/sdk @ardrive/turbo-sdk @story-protocol/core-sdk viem pinata-web3 dotenv
```

Create a `.env` file and add your credentials:

```text .env
# Story Protocol credentials
PRIVATE_KEY=<your_ethereum_private_key>

# Pinata (optional for IPFS)
PINATA_JWT=<your_pinata_jwt>

# Arweave wallet path (or you can load directly in code)
ARWEAVE_WALLET_PATH=./wallet.json

# AR.IO ANT process ID
ANT_PROCESS_ID=<your_ant_process_id>
```

Next, let's set up the Arweave and AR.IO clients:

```typescript arweave-utils.ts
import fs from 'fs';
import { ANT, ArweaveSigner } from '@ar.io/sdk';
import { TurboFactory, ArweaveSigner as TurboSigner } from '@ardrive/turbo-sdk';
import dotenv from 'dotenv';

dotenv.config();

// Load the Arweave wallet
const getArweaveWallet = () => {
  if (process.env.ARWEAVE_WALLET_PATH) {
    return JSON.parse(fs.readFileSync(process.env.ARWEAVE_WALLET_PATH, 'utf8'));
  }
  throw new Error("Arweave wallet not found. Please set ARWEAVE_WALLET_PATH in .env");
};

const jwk = getArweaveWallet();

// Initialize the AR.IO ANT client
export const ant = ANT.init({
  signer: new ArweaveSigner(jwk),
  processId: process.env.ANT_PROCESS_ID || "",
});

// Initialize the Turbo client for uploads
export const turbo = TurboFactory.init({
  signer: new TurboSigner(jwk)
});

// Helper function to determine content type based on file extension
export function getContentType(fileName) {
  const extension = fileName.split('.').pop().toLowerCase();
  const contentTypes = {
    'jpg': 'image/jpeg',
    'jpeg': 'image/jpeg',
    'png': 'image/png',
    'gif': 'image/gif',
    'pdf': 'application/pdf',
    'json': 'application/json',
    'html': 'text/html',
    'txt': 'text/plain',
    // Add more types as needed
  };
  
  return contentTypes[extension] || 'application/octet-stream';
}
```

## 3. Upload assets to Arweave

Now let's add the functionality to upload files to Arweave:

```typescript main.ts
// ...previous code

async function uploadToArweave(filePath) {
  // Get the file name from the path
  const fileName = filePath.split('/').pop();
  
  // Read the file
  const fileData = fs.readFileSync(filePath);
  
  // Determine content type based on file extension
  const contentType = getContentType(fileName);

  // Upload the file to Arweave
  const uploadResult = await turbo.uploadFile({

    tags: [
      { name: 'Content-Type', value: contentType },
      { name: 'App-Name', value: 'Story-Arweave Integration' },
    ]
  });
  
  console.log(`File uploaded with transaction ID: ${uploadResult.id}`);
  return uploadResult.id;
}
```

## 4. Create IPFS metadata for Story Protocol

For Story Protocol compatibility, we'll also create metadata on IPFS:

```typescript upload-ipfs.ts
import { PinataSDK } from "pinata-web3";
import dotenv from "dotenv";

dotenv.config();

const pinata = new PinataSDK({
  pinataJwt: process.env.PINATA_JWT,
});

export async function uploadJSONToIPFS(jsonMetadata: any): Promise<string> {
  if (!process.env.PINATA_JWT) {
    throw new Error("Missing PINATA_JWT environment variable");
  }
  
  const { IpfsHash } = await pinata.upload.json(jsonMetadata);
  return IpfsHash;
}
```

## 10. Adding License Terms

Now that your IP is registered on Story Protocol and accessible via Arweave and AR.IO, you can create and attach license terms to it:

```typescript
// Example of adding license terms (run this after registering the IP)
async function addLicenseTerms(ipId) {
  // Create PIL terms
  const createTermsResponse = await client.licensing.createPILTerms({
    commercial: true,
    derivatives: true,
    reciprocal: false,
    attribution: true,
    txOptions: { waitForTransaction: true },
  });
  
  console.log(`Created PIL terms with ID: ${createTermsResponse.pilId}`);
  
  // Attach terms to IP
  const attachTermsResponse = await client.licensing.attachPILTerms({
    pilId: createTermsResponse.pilId,
    ipId,
    txOptions: { waitForTransaction: true },
  });
  
  console.log(`Attached PIL terms with transaction hash: ${attachTermsResponse.txHash}`);
  return createTermsResponse.pilId;
}

// Add this to your main function
const pilId = await addLicenseTerms(ipId);
```
