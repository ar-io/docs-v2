# How to Access Data on Arweave

Once data is stored on Arweave, you need to know how to access it. Choose the method that best fits your needs.

## Access Methods

<Cards>
  <Card href="/how-to-access-data/http-requests" title="HTTP Requests">
    **Perfect for:** Simple data retrieval, web applications, direct file access
    
    **Examples:** arweave.net, arweave.world, g8way.io
    
    **Pros:**
    - Simple URL-based access
    - Works in any web browser
    - No additional setup required
    
    **Cons:**
    - Limited querying capabilities
    - No metadata filtering
    - Basic functionality only
  </Card>

{" "}

<Card href="/how-to-access-data/graphql-queries" title="GraphQL Queries">
  **Perfect for:** Complex queries, metadata filtering, application development
  **Examples:** Goldsky, ar-gql library, custom GraphQL endpoints **Pros:** -
  Powerful querying capabilities - Filter by tags and metadata - Flexible data
  retrieval **Cons:** - Requires GraphQL knowledge - More complex than HTTP -
  Gateway-dependent
</Card>

{" "}

<Card href="/how-to-access-data/arns" title="ArNS (Arweave Name System)">
  **Perfect for:** Human-readable URLs, domain management, permanent addressing
  **Examples:** myapp.arweave.net, project.arweave.net **Pros:** -
  Human-readable URLs - Permanent addressing - Easy to remember **Cons:** -
  Requires ARNS registration - Additional cost - Name management needed
</Card>

  <Card href="/how-to-access-data/wayfinder" title="Wayfinder">
    **Perfect for:** Decentralized discovery, content routing, network optimization
    
    **Examples:** AR.IO Wayfinder, custom routing solutions
    
    **Pros:**
    - Decentralized discovery
    - Network optimization
    - Advanced routing
    
    **Cons:**
    - More complex setup
    - Requires understanding of routing
    - Newer technology
  </Card>
</Cards>

## Why Access Methods Matter

Different access methods serve different purposes:

- **HTTP Requests** - Direct, simple access to any transaction
- **GraphQL Queries** - Powerful searching and filtering capabilities
- **ArNS** - Human-readable, permanent addressing
- **Wayfinder** - Decentralized, optimized content discovery

## Ready to Access Data?

**Start with HTTP Requests** - the simplest way to access your data.

**Need advanced querying?** Try [GraphQL Queries](/how-to-access-data/graphql-queries) for powerful search capabilities.

**Want human-readable URLs?** Explore [ArNS](/how-to-access-data/arns) for permanent addressing.
