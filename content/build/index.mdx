# Build with AR.IO

This section is tailored for builders and developers looking to build with AR.IO. It includes introductory information on the ways in which to build, as well as reference docs to SDKs and APIs, and guides for common use cases.

## Start Building Today

    <Steps>
      <Step>
        ### Choose Your SDK

        Pick the SDK that matches your use case:
        - **AR.IO SDK** - Interact with the AR.IO Network and ArNS
        - **Turbo SDK** - Upload data with fast, reliable service
        - **Wayfinder SDK** - Access Arweave data through decentralized gateways
      </Step>
      <Step>
        ### Install and Configure

        ```bash
        npm install @ar.io/sdk
        # or
        npm install @ardrive/turbo-sdk
        # or
        npm install @ar.io/wayfinder-core
        ```
      </Step>
      <Step>
        ### Build Your First App

        Follow our guides to create your first permanent web application:
        - Upload data to Arweave
        - Resolve ArNS names
        - Access decentralized data
      </Step>
    </Steps>

  </Tab>

## Where to Start

### For Web Developers

**Recommended Path:** Start with the **Turbo SDK** for easy uploads and the **AR.IO SDK** for ArNS integration.

1. Follow the Quick Start guide above
2. Deploy a static website with ArNS
3. Add dynamic data storage capabilities

### For Backend Developers

**Recommended Path:** Use the **AR.IO SDK** for network interactions and consider running a gateway.

1. Integrate AR.IO SDK into your backend
2. Implement data archival workflows
3. Set up gateway infrastructure

### For dApp Developers

**Recommended Path:** Combine **Wayfinder SDK** for decentralized access with **AR.IO SDK** for full functionality.

1. Use Wayfinder for censorship-resistant data access
2. Implement ArNS for user-friendly naming
3. Build fully decentralized applications
