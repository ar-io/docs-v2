# Access Data with HTTP Requests

The simplest way to access data on Arweave is through **HTTP requests** to gateways. This method works in any web browser and requires no additional setup.

## Are You Building an Application?

<Callout type="info">
  **For Developers:** If you're building an application that needs to discover
  data, check out [GraphQL Queries](/how-to-access-data/graphql-queries) for
  powerful search capabilities. HTTP requests are perfect for accessing data
  when you already know the transaction ID.
</Callout>

## What are AR.IO Gateways?

[AR.IO Gateways](https://cookbook.bobinstein.com/fundamentals/accessing-arweave-data/gateways.html) serve as the interface between the Arweave network and end-users, making permaweb data easily accessible through standard web browsers.

**Key Functions:**

- **Content Caching** - Store frequently accessed transactions
- **Data Indexing** - Provide search and query capabilities
- **Network Seeding** - Help distribute transactions
- **Content Moderation** - Apply content policies

## Basic Access Pattern

Access any transaction using this URL structure:

```
https://<gateway>/<transaction-id>
```

**Examples:**

- `https://arweave.net/bVLEkL1SOPFCzIYi8T_QNnh17VlDp4RylU6YTwCMVRw`
- `https://permagate.io/FguFk5eSth0wO8SKfziYshkSxeIYe7oK9zoPN2PhSc0`

## Popular Gateways

- **arweave.net** - Operated by the AR.IO team
- **permagate.io** - Community gateway
- **g8way.io** - Community gateway
- **arweave.developerdao.com** - Community gateway

## AR.IO Gateway Security

AR.IO gateways implement security measures by redirecting requests to sandbox subdomains:

**Why Redirects Happen:**

- **Security Isolation** - Content is served from isolated sandbox environments
- **CSP Protection** - Prevents cross-site scripting attacks
- **Resource Isolation** - Limits potential security vulnerabilities

**What to Expect:**

- Initial request: `https://arweave.net/transaction-id`
- Redirects to: `https://sandbox.arweave.net/transaction-id` (or similar)
- Final content served from sandbox subdomain

**Important:** Always follow redirects in your applications - the final sandbox URL contains the actual content.

## Using in Applications

**JavaScript Example with Axios:**

```js
import axios from "axios";

// Fetch data from Arweave (follows redirects automatically)
const response = await axios.get("https://arweave.net/your-transaction-id", {
  maxRedirects: 5, // Follow up to 5 redirects
  validateStatus: function (status) {
    return status >= 200 && status < 300; // Accept redirects
  },
});

console.log(response.data);
```

**Handling AR.IO Gateway Redirects:**

AR.IO gateways redirect to sandbox subdomains for security. Axios handles this automatically:

```js
import axios from "axios";

const fetchArweaveData = async (transactionId) => {
  try {
    // AR.IO gateways will redirect to sandbox subdomain
    const response = await axios.get(`https://arweave.net/${transactionId}`, {
      maxRedirects: 5,
      timeout: 10000, // 10 second timeout
    });

    return response.data;
  } catch (error) {
    console.error("Error fetching data:", error.message);
    throw error;
  }
};

// Usage
const data = await fetchArweaveData("your-transaction-id");
```

**HTML Example:**

```html
<!-- Direct image access (browsers handle redirects automatically) -->
<img src="https://arweave.net/your-image-id" alt="My Image" />
```

## Path Manifests

For organized file collections, use [Path Manifests](https://cookbook.bobinstein.com/fundamentals/accessing-arweave-data/manifests.html):

```
https://arweave.net/<manifest-id>/path/to/file
```

**Example:**

- `https://arweave.net/X8Qm…AOhA/index.html`
- `https://arweave.net/X8Qm…AOhA/styles.css`
- `https://arweave.net/X8Qm…AOhA/assets/logo.png`

## Why Choose HTTP Requests?

HTTP requests provide the most straightforward way to access Arweave data.

<Callout type="note">
  **Perfect for:** Simple data retrieval, web applications, direct file access,
  and any use case where you already know the transaction ID.
</Callout>

| Feature              | HTTP Requests      | GraphQL Queries            | ArNS                  | Wayfinder                |
| -------------------- | ------------------ | -------------------------- | --------------------- | ------------------------ |
| **Setup Complexity** | None required      | Requires GraphQL knowledge | Requires registration | Requires SDK integration |
| **Use Case**         | Direct access      | Data discovery             | Human-readable URLs   | Production apps          |
| **Performance**      | Fast for known IDs | Slower for complex queries | Fast once set up      | Optimized routing        |
| **Flexibility**      | Basic              | High                       | Medium                | High                     |
| **Dependencies**     | None               | Gateway-dependent          | ArNS service          | Wayfinder SDK            |

<Callout type="info">
  **Need help deciding?** Most users should start with HTTP requests for simple
  access. Only consider alternatives if you need data discovery, human-readable
  URLs, or production-grade reliability.
</Callout>
